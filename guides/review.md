# Code Review

Cross-stack code review approach at Bytetalent. The canonical checklist with all categories, severity rubric, and output format lives in the skill:

**Code review checklist**: see [`skills/code/code-review-checklist`](../skills/code/code-review-checklist/SKILL.md).

---

## How to apply the checklist

Work through the six checklist categories systematically. For each finding, report: **severity** (Critical / High / Medium / Low), **file path with line number**, **what's wrong**, and **how to fix it**.

Organize findings into a prioritized table grouped by severity. After the table, provide a suggested implementation order that addresses Critical/High items first.

---

## Cadence

Reviews land as dated files (`docs/app-review-YYYY-MM-DD.md` or equivalent) in the consuming repo. Open action items from the latest review get tracked in that repo's plan file.
